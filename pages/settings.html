使用者
狀態切換為「停用即除役」，不提供硬刪除；停用後之登入請求一律回 403（REQ-AUTH-005）。連續登入失敗 5 次帳號將鎖定 15 分鐘（429，REQ-AUTH-002）；管理員可於本表直接解鎖或代為重設密碼（REQ-AUTH-004）。
角色
點列展開「模組 × 權限」matrix；權限變更於使用者下次請求生效，並寫入審計（REQ-RBAC-007/009）。停用角色不可再指派給帳號；已持有者依 REQ-RBAC-008 於下次請求起失去該角色權限（其他角色不受影響）。內建 super admin 角色不可停用。
Break-glass 自助重設
super admin 綁定 email 可自助觸發密碼重設，不需其他管理員介入；完成後系統自動寫入審計（動作／時間／來源 IP）， 並即時通知其他 super admin。
外部憑證
憑證以 AES-256 加密儲存；密碼欄位永不回顯明文，僅顯示遮罩，更新一律寫入審計。
API Token
token 綁最小 scope、可設效期、可即時撤銷；憑證讀取 scope 另綁 IP 白名單（限排程主機）＋限流＋每次讀取寫審計，白名單外拒絕並告警（REQ-IMP-013）。
append-only：任何角色（含 super admin）皆不可刪除或修改審計日誌，僅可查詢與匯出。

原型說明：本頁為互動原型（零依賴、假資料，不落地至 localStorage 之 CRM 主資料層）。帳號／角色／憑證／Token／審計等清單為頁面內示意資料，重新整理將還原初始狀態； 「儲存設定」「更新憑證」等動作以 toast 示意寫入審計，正式版將串接真實後端與 append-only 審計儲存。姓名全站一律遮罩（陳○明）。 車隊 → 車隊地區對照為匯入唯讀字典，本頁不提供人工新增／修正。 行動電話僅存在於客服模組，本頁不顯示電話欄位。

更新憑證

憑證將以 AES-256 加密儲存；儲存後密碼欄位永不回顯明文。

新增角色

新角色預設不具任何權限；建立後請於角色列表點列展開，逐一勾選模組權限。

新增活動類型

新類型預設啟用，立即出現於「隊員活動」建立時之類型選單。

新增帳號

帳號由 Supabase Auth 以 email＋password 管理；設定預設密碼交付新使用者，首次登入將強制變更密碼。

重設密碼確認

將寄送密碼重設信至該帳號綁定信箱，原密碼即刻失效；此動作將寫入審計日誌。

此帳號具 super admin 角色：正式版依 REQ-AUTH-004，代為重設密碼僅限「其他 super admin」執行；本原型不模擬登入身分，不擋此操作。

建立 Token

token 綁最小 scope、可設效期、可即時撤銷；scope 值域沿用 RBAC 權限 key（正式版專屬 scope 定義待訂）。
